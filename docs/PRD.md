# 바이브 코딩 표준 가이드 PRD

## 1. 개요

### 1.1 프로젝트명
**Vibe Coding Recommendations** - 바이브 코딩 표준 가이드

### 1.2 목적
AI 기반 코딩 도구(Claude Code, Cursor, GitHub Copilot 등)를 활용한 "바이브 코딩" 방법론에 대한 표준 가이드를 제공한다. 개발자들이 AI 도구를 효과적으로 활용하여 생산성을 높이고, 일관된 품질의 코드를 작성할 수 있도록 돕는다.

### 1.3 배경
- AI 코딩 도구의 급속한 발전으로 개발 방식이 변화하고 있음
- "바이브 코딩"이라는 새로운 개발 패러다임 등장
- 체계적인 가이드 부재로 인한 품질 편차 발생
- 조직 내 표준화된 AI 도구 활용 방법론 필요

---

## 2. 목표

### 2.1 핵심 목표
1. **표준화**: 바이브 코딩에 대한 일관된 가이드라인 제공
2. **교육**: 세션 발표를 통한 지식 전파
3. **접근성**: GitHub Pages를 통한 문서 공개 배포

### 2.2 성공 지표
- 가이드 문서 완성도 (핵심 주제 커버율)
- 발표 슬라이드 품질 (Marp 형식)
- GitHub Pages 배포 완료 및 접근 가능성

---

## 3. 범위

### 3.1 포함 범위 (In Scope)

#### 문서 콘텐츠
- 바이브 코딩 개념 정의 및 원칙
- AI 코딩 도구 비교 및 선택 가이드
- 효과적인 프롬프트 작성법
- 코드 리뷰 및 품질 관리
- 보안 고려사항
- 팀 협업 시 베스트 프랙티스
- 케이스 스터디 및 예제

#### 발표 자료
- Marp 형식 슬라이드
- 핵심 내용 요약
- 시각적 다이어그램 (흑백 Mermaid)

#### 배포
- GitHub Pages 정적 사이트
- 문서 네비게이션
- 슬라이드 PDF 다운로드

### 3.2 제외 범위 (Out of Scope)
- 특정 도구의 상세 튜토리얼
- 실시간 협업 기능
- 사용자 인증/권한 관리
- 다국어 지원 (초기 버전)

## 4. 산출물

### 4.1 디렉토리 구조

```
vibe-coding-recommendations/
├── README.md                 # 프로젝트 소개
├── docs/
│   ├── PRD.md               # 본 문서
│   ├── guide.md             # 가이드 문서 (단일 파일)
│   └── slides/              # Marp 슬라이드
│       ├── main.md
│       └── assets/
├── .github/
│   └── workflows/
│       └── deploy.yml       # GitHub Pages 배포
└── package.json             # Marp CLI 등 의존성
```

### 4.2 가이드 문서 구성

단일 파일(`guide.md`) 내에 다음 섹션들을 포함한다. 콘텐츠가 충분히 확장되면 개별 파일로 분리할 수 있다.

- 이 문서의 목적
- 이 문서의 저자는 30년 경력을 가진 개발자이며, 현재 한국에서 AI 트렌스포메이션 컨설팅과 교육을 하고 있으며, AI 트렌스포메이션의 유용성을 스스로 입증하기 위해 자신의 서비스 개발도 수행하고 있다.
- 바이브 코딩의 정의
  - 현 시점의 바이브 코딩은 단순히 자연어로 코드를 생성하는것이 아니라, AI 도구를 최대한 활용하여 개발 생산성을 극대화하는 방법론을 총칭한다.
- 핵심 원칙
- 마인드셋
- 안티패턴
- 도구
- 프롬프팅
- 워크플로우
- 품질 관리
- 보안
- 협업
- 사례 연구

### 4.3 슬라이드 구성

Marp 형식으로 작성하며, 다음 섹션 포함:
- 타이틀 슬라이드
- 아젠다
- 각 장별 핵심 요약 (2-3 슬라이드)
- Q&A
- 참고 자료
